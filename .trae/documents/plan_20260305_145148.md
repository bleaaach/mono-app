## 修复标签滑动与页面切换手势冲突

### 问题分析
- 标签筛选区域使用 `ScrollView` 水平滑动
- 外层使用 `PanGestureHandler` 处理页面切换
- 两者手势冲突，导致滑动标签时触发页面切换

### 解决方案
1. **使用 `react-native-gesture-handler` 的 ScrollView**
   - 替换 React Native 原生 ScrollView
   - 它能更好地处理嵌套手势

2. **为标签区域添加手势拦截**
   - 使用 `simultaneousHandlers` 或 `waitFor` 属性
   - 让标签滑动优先于页面切换

3. **修改文件**
   - `src/screens/DiaryScreen.tsx` - 修复标签区域手势冲突
   - `src/screens/TodoScreen.tsx` - 同样的问题
   - `src/screens/HabitScreen.tsx` - 同样的问题
   - `src/screens/TimeScreen.tsx` - 同样的问题

### 实现步骤
1. 导入 `ScrollView` from `react-native-gesture-handler`
2. 为外层 `PanGestureHandler` 添加 `simultaneousHandlers` 配置
3. 确保标签区域能独立滑动
