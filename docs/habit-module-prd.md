## 一、模块概述

### 1.1 产品定位
习惯打卡模块是一款帮助用户建立、追踪和维护良好习惯的工具。通过可视化的数据展示、成就系统和热力图，激励用户持续坚持，最终实现习惯养成。

### 1.2 目标用户
- 希望养成良好生活习惯的人群
- 需要自我管理和提升的用户
- 追求持续成长的自律型用户

### 1.3 核心价值
- **简单易用**：一键打卡，无复杂操作
- **数据可视化**：热力图、统计图表直观展示进度
- **正向激励**：成就系统、连续天数激励持续坚持
- **灵活定制**：支持多种频率和目标类型

## 三、数据模型

### 3.1 习惯 (Habit)

```typescript
interface Habit {
  id: string;                    // 唯一标识
  name: string;                  // 习惯名称
  description?: string;          // 描述
  color: string;                 // 颜色标记
  icon: string;                  // 图标
  category: string;              // 分类ID
  tags: string[];                // 标签
  frequency: {                   // 频率配置
    type: 'daily' | 'weekly' | 'monthly' | 'custom';
    days?: number[];             // 指定日期
    interval?: number;           // 间隔天数
  };
  target: {                      // 目标配置
    type: 'streak' | 'count' | 'none';
    value?: number;              // 目标值
    deadline?: string;           // 截止日期
  };
  reminders: {                   // 提醒配置
    time: string;
    enabled: boolean;
  }[];
  createdAt: string;             // 创建时间
  archived: boolean;             // 是否归档
  paused: boolean;               // 是否暂停
  order: number;                 // 排序
}
```

### 3.2 打卡记录 (HabitLog)

```typescript
interface HabitLog {
  id: string;                    // 记录ID
  habitId: string;               // 关联习惯ID
  date: string;                  // 打卡日期 YYYY-MM-DD
  completed: boolean;            // 是否完成
  count: number;                 // 打卡次数
  note?: string;                 // 备注
  isMakeup: boolean;             // 是否补打卡
  createdAt: string;             // 创建时间
}
```

### 3.3 统计数据 (HabitStats)

```typescript
interface HabitStats {
  habitId: string;
  totalCompletions: number;      // 总完成次数
  currentStreak: number;         // 当前连续天数
  longestStreak: number;         // 最长连续天数
  completionRate: number;        // 完成率
  weeklyStats: number[];         // 本周每日统计
  monthlyStats: number[];        // 本月每日统计
}
```

## 四、后续功能规划

### 4.1 Phase 2: SOP/Routine 功能（优先级：高）

#### 4.1.1 习惯模板库
**需求描述**：提供预置的习惯模板，用户可一键添加常见习惯。

**模板示例**：
- 🌅 早晨例行：早起、喝水、晨练、冥想、阅读
- 🌙 晚间例行：复盘、阅读、早睡
- 💪 健康生活：喝水8杯、运动30分钟、早睡早起
- 📚 学习成长：阅读、背单词、学习技能
- 🧘 身心健康：冥想、瑜伽、感恩日记

**功能点**：
- 模板浏览和预览
- 一键添加模板中的所有习惯
- 自定义修改模板内容
- 用户自建模板并分享

#### 4.1.2 习惯套件 (Routine)
**需求描述**：将多个习惯组合成一个套件，支持批量操作。

**功能点**：
- 创建习惯套件（如"早晨7件事"）
- 套件内习惯排序
- 一键开始套件（依次提醒打卡）
- 套件完成进度追踪
- 套件模板市场

#### 4.1.3 时间段管理
**需求描述**：为习惯分配执行时间段，形成日程化习惯。

**功能点**：
- 为习惯设置建议执行时间
- 按时间段分组展示（早晨/上午/下午/晚间）
- 时间段完成度统计
- 智能推荐最佳执行时间

### 4.2 Phase 3: 提醒系统（优先级：高）

#### 4.2.1 本地推送通知
**功能点**：
- 设置提醒时间
- 多时间点提醒

#### 4.2.2 智能提醒

#### 4.3.2 多次打卡
**功能点**：
- 支持每日多次打卡
- 设置每日目标次数
- 打卡计数器

#### 4.3.3 打卡备注
**功能点**：
- 打卡时添加文字备注
- 打卡时添加图片
- 打卡心情记录
- 备注历史查看

### 4.4 Phase 5: 高级统计（优先级：中）

## 八、风险与挑战

### 8.1 技术风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AsyncStorage性能瓶颈 | 大量数据时性能下降 | 考虑迁移到SQLite或Realm |
| 推送通知权限 | 用户拒绝则无法提醒 | 引导用户开启，提供应用内提醒 |
| 跨平台兼容性 | 部分功能iOS/Android差异 | 充分测试，做好降级方案 |

### 8.2 产品风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 功能过载 | 用户学习成本高 | 渐进式展示，核心功能优先 |
| 用户流失 | 新鲜感过后放弃 | 持续激励，社交功能增强粘性 |
| 数据丢失 | 用户信任度下降 | 完善备份机制，错误处理 |

## 九、成功指标

### 9.1 核心指标
- **日活跃用户 (DAU)**：每日打开App的用户数
- **打卡完成率**：每日打卡数/应打卡数
- **7日留存率**：新用户7天后仍活跃的比例
- **平均连续天数**：用户平均连续打卡天数

### 9.2 目标值
| 指标 | 1个月目标 | 3个月目标 |
|------|-----------|-----------|
| DAU | 100 | 500 |
| 打卡完成率 | 60% | 75% |
| 7日留存率 | 40% | 50% |
| 平均连续天数 | 5天 | 10天 |
