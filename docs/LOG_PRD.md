# 生活日志 (Life Log) 功能产品需求文档

## 1. 产品概述

### 1.1 产品定位
生活日志是一个全能型生活记录系统，类似于 anylog、lifelog、marktimes、marknow 等应用。它允许用户以多种格式记录生活中的各种事件、状态和数据，构建个人生活的时间线档案。

### 1.2 目标用户
- 希望全面记录生活的人群
- 需要追踪健康、情绪、习惯的用户
- 想要分析生活模式的数据爱好者
- 需要记录工作/学习时间的效率追求者

### 1.3 核心价值
- **全能记录**：支持 8 种日志类型，覆盖生活方方面面
- **灵活标签**：多维度分类，便于检索和分析
- **时间线视图**：按时间顺序展示所有记录
- **数据统计**：提供基础统计和趋势分析

## 2. 功能需求

### 2.1 日志类型系统

#### 2.1.1 事件 (Event)
- **用途**：记录发生的任何事情
- **字段**：标题、内容、时间、分类、标签
- **示例**："参加了朋友的婚礼"、"完成了项目交付"

#### 2.1.2 计数 (Count)
- **用途**：记录可量化的次数
- **字段**：标题、数值、单位、时间、分类、标签
- **示例**："喝水 8 杯"、"俯卧撑 50 个"

#### 2.1.3 计时 (Timer)
- **用途**：记录活动持续时间
- **字段**：标题、开始时间、结束时间、时长、分类、标签
- **功能**：内置计时器，支持实时计时
- **示例**："阅读 45 分钟"、"冥想 20 分钟"

#### 2.1.4 心情 (Mood)
- **用途**：记录情绪状态
- **字段**：标题、心情类型、内容、时间、标签
- **心情选项**：
  - 😊 开心 - 积极愉悦的状态
  - 🤩 兴奋 - 充满活力和热情
  - 😌 平静 - 内心安宁
  - 😴 疲惫 - 身体或精神疲劳
  - 😢 难过 - 悲伤或失落
  - 😠 生气 - 愤怒或不满
  - 😰 焦虑 - 紧张或担忧

#### 2.1.5 笔记 (Note)
- **用途**：纯文本记录
- **字段**：标题、内容、时间、分类、标签
- **示例**：灵感记录、会议纪要、读书笔记

#### 2.1.6 支出 (Expense)
- **用途**：记录消费支出
- **字段**：标题、金额、货币单位、时间、分类、标签
- **示例**："午餐 35 元"、"买书 89 元"

#### 2.1.7 健康 (Health)
- **用途**：记录健康相关数据
- **字段**：标题、数值、单位、时间、分类、标签
- **示例**："体重 65kg"、"血压 120/80"

#### 2.1.8 位置 (Location)
- **用途**：记录地理位置信息
- **字段**：标题、经纬度、地址、时间、标签
- **状态**：预留功能，后续可集成地图

## 4. 技术规范

### 4.1 数据模型

```typescript
// 日志类型
export type LogEntryType = 'event' | 'count' | 'timer' | 'mood' | 'note' | 'expense' | 'health' | 'location';

// 日志条目
export interface LogEntry {
  id: string;                    // 唯一标识
  type: LogEntryType;            // 日志类型
  title: string;                 // 标题
  content?: string;              // 内容
  value?: number;                // 数值
  unit?: string;                 // 单位
  tags: string[];                // 标签
  category: string;              // 分类ID
  location?: {                   // 位置信息
    latitude: number;
    longitude: number;
    address?: string;
  };
  startTime?: string;            // 开始时间（ISO）
  endTime?: string;              // 结束时间（ISO）
  duration?: number;             // 时长（秒）
  mood?: 'happy' | 'excited' | 'neutral' | 'tired' | 'sad' | 'angry' | 'anxious';
  images?: string[];             // 图片URL列表
  createdAt: string;             // 创建时间（ISO）
  updatedAt: string;             // 更新时间（ISO）
}

// 日志分类
export interface LogCategory {
  id: string;
  name: string;
  color: string;
  icon: string;
  type: LogEntryType;
  unit?: string;
}
```

### 4.2 存储方案

使用 AsyncStorage 进行本地存储：

| 键名 | 数据类型 | 说明 |
|------|----------|------|
| @mono:log_entries | LogEntry[] | 日志条目列表 |
| @mono:log_categories | LogCategory[] | 自定义分类 |
| @mono:log_tags | string[] | 所有标签 |

### 4.3 存储操作

```typescript
export const logStorage = {
  get: () => Promise<LogEntry[] | null>,
  set: (value: LogEntry[]) => Promise<void>,
  getByDate: (date: string) => Promise<LogEntry[]>,
  getByType: (type: string) => Promise<LogEntry[]>,
  getByCategory: (category: string) => Promise<LogEntry[]>,
  getByTag: (tag: string) => Promise<LogEntry[]>,
  search: (keyword: string) => Promise<LogEntry[]>,
};
```

## 5. 用户交互流程

### 5.1 添加日志流程

```
1. 用户点击右上角 [+] 按钮
   ↓
2. 弹出添加日志模态框
   ↓
3. 用户选择日志类型（默认：事件）
   ↓
4. 用户填写标题（必填）
   ↓
5. 根据类型显示对应字段
   - 计数/支出/健康：显示数值+单位输入
   - 计时：显示计时器控件
   - 心情：显示心情选择
   ↓
6. 用户选择分类和标签
   ↓
7. 用户点击"保存日志"
   ↓
8. 验证标题不为空
   ↓
9. 保存到 AsyncStorage
   ↓
10. 刷新列表显示新日志
```

### 5.2 筛选流程

```
1. 用户点击"筛选"按钮
   ↓
2. 弹出筛选模态框
   ↓
3. 用户选择筛选条件（可多选）
   ↓
4. 实时应用筛选到列表
   ↓
5. 用户可点击"清除所有筛选"重置
```

### 5.3 删除流程

```
1. 用户在日志卡片上点击"删除"
   ↓
2. 立即从列表移除（无二次确认，简化操作）
   ↓
3. 更新 AsyncStorage
```
