# 收纳模块 (Storage) 产品需求文档

## 1. 产品概述

### 1.1 产品定位
收纳模块是一个智能物品管理系统，帮助用户数字化管理家中各类物品。通过空间-物品-分类的三维管理体系，解决"东西放哪儿了"的痛点，让收纳变得简单高效。

### 1.2 目标用户
- 物品繁多、经常找不到东西的都市人群
- 喜欢整理收纳、追求生活品质的精致生活者
- 搬家/装修需要物品清点的用户
- 有收藏爱好（书籍、手办、美妆等）的爱好者
- 需要管理家庭物品的家庭主妇/主夫

### 1.3 核心价值
- **空间可视化**：直观展示物品所在位置，支持多级空间嵌套
- **智能检索**：多维度筛选（空间/分类/标签），快速定位物品
- **过期提醒**：食品、药品、化妆品保质期预警
- **数据统计**：物品总量、价值统计、分类占比分析
- **借还管理**：记录物品借出/归还状态

## 2. 功能需求

### 2.1 空间管理系统

#### 2.1.1 空间层级结构
支持无限层级嵌套的空间管理：

```
家
├── 客厅
│   ├── 电视柜
│   │   ├── 第一层抽屉
│   │   └── 第二层抽屉
│   └── 茶几
├── 卧室
│   ├── 衣柜
│   │   ├── 上层
│   │   ├── 挂衣区
│   │   └── 抽屉
│   └── 床头柜
├── 厨房
│   ├── 冰箱
│   │   ├── 冷藏室
│   │   └── 冷冻室
│   └── 橱柜
└── 书房
    └── 书架
```

#### 2.1.2 空间属性
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ID | string | 是 | 唯一标识 |
| 名称 | string | 是 | 空间名称 |
| 父级ID | string | 否 | 上级空间ID，根节点为空 |
| 层级 | number | 是 | 当前层级深度 |
| 路径 | string | 是 | 完整路径（如：家/客厅/电视柜）|
| 图标 | string | 否 | 空间图标 |
| 颜色 | string | 否 | 标识颜色 |
| 图片 | string[] | 否 | 空间照片 |
| 备注 | string | 否 | 空间描述 |
| 排序 | number | 是 | 同级排序 |

#### 2.1.3 空间操作
- **创建空间**：支持选择父级空间，自动计算层级
- **编辑空间**：修改名称、图标、颜色、图片
- **删除空间**：删除前需确认，可选择迁移物品或一并删除
- **移动空间**：拖拽调整空间位置或层级
- **展开/折叠**：树形结构支持展开折叠

### 2.2 物品管理系统

#### 2.2.1 物品基础信息
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ID | string | 是 | 唯一标识 |
| 名称 | string | 是 | 物品名称 |
| 空间ID | string | 是 | 所属空间 |
| 分类ID | string | 是 | 物品分类 |
| 图片 | string[] | 否 | 物品照片（最多9张）|
| 数量 | number | 否 | 物品数量，默认1 |
| 单位 | string | 否 | 计量单位（个/件/瓶/盒等）|
| 品牌 | string | 否 | 品牌信息 |
| 型号 | string | 否 | 型号/规格 |
| 价格 | number | 否 | 购买价格 |
| 货币 | string | 否 | 货币单位，默认CNY |
| 购买日期 | string | 否 | ISO日期格式 |
| 购买渠道 | string | 否 | 购买来源 |
| 保质期 | string | 否 | 食品/药品/化妆品过期日期 |
| 保修期 | string | 否 | 保修截止日期 |
| 标签 | string[] | 否 | 自定义标签 |
| 备注 | string | 否 | 物品描述 |
| 状态 | enum | 是 | 正常/借出/维修/丢弃 |
| 创建时间 | string | 是 | ISO时间戳 |
| 更新时间 | string | 是 | ISO时间戳 |

#### 2.2.2 物品状态管理
| 状态 | 说明 | 图标 |
|------|------|------|
| 正常 | 物品在库可用 | ✅ |
| 借出 | 物品已借给他人 | 📤 |
| 维修 | 物品正在维修中 | 🔧 |
| 丢弃 | 物品已丢弃/丢失 | 🗑️ |

#### 2.2.3 借还管理
当物品状态为"借出"时，记录借还信息：
| 字段 | 类型 | 说明 |
|------|------|------|
| 借出对象 | string | 借给谁 |
| 借出日期 | string | 借出时间 |
| 预计归还 | string | 预计归还日期 |
| 实际归还 | string | 实际归还时间 |
| 借出备注 | string | 借出说明 |

- 支持分类图标和颜色设置

### 2.4 标签系统

#### 2.4.1 预设标签
- **重要性**：重要、常用、备用、闲置
- **状态**：全新、九成新、二手、损坏
- **季节**：春、夏、秋、冬、四季
- **场景**：工作、学习、旅行、运动、居家
- **特征**：易碎、贵重、 bulky（大件）、需冷藏

#### 2.4.2 标签功能
- 支持多标签关联
- 支持自定义标签创建
- 标签云展示
- 按标签快速筛选

### 2.5 搜索与筛选

#### 2.5.1 关键词搜索
- 搜索范围：物品名称、品牌、型号、备注
- 实时搜索，即时反馈
- 搜索历史记录
- 热门搜索推荐

#### 2.5.2 高级筛选
| 筛选维度 | 说明 |
|----------|------|
| 空间 | 选择特定空间及子空间 |
| 分类 | 单选或多选分类 |
| 标签 | 选择标签组合 |
| 状态 | 正常/借出/维修/丢弃 |
| 保质期 | 即将过期/已过期/自定义日期范围 |
| 价格区间 | 自定义价格范围 |
| 购买日期 | 最近7天/30天/自定义范围 |

#### 2.5.3 排序方式
- 最近添加（默认）
- 名称排序（A-Z）
- 保质期临近
- 价格高低
- 购买时间

### 2.6 过期提醒系统

#### 2.6.1 提醒类型
| 类型 | 触发条件 | 提醒方式 |
|------|----------|----------|
| 即将过期 | 保质期前7天/30天 | 应用内红点+推送 |
| 已过期 | 超过保质期 | 应用内警告标记 |
| 借出到期 | 超过预计归还日期 | 应用内提醒 |
| 保修到期 | 保修期前30天 | 应用内提醒 |

#### 2.6.2 提醒设置
- 可设置提前提醒天数
- 可关闭特定类型提醒
- 支持批量标记已处理

### 2.7 统计功能

#### 2.7.1 基础统计
| 统计项 | 说明 |
|--------|------|
| 物品总数 | 所有物品数量总和 |
| 空间数量 | 空间节点总数 |
| 分类数量 | 使用的分类数 |
| 总价值 | 所有物品价格总和 |
| 本月新增 | 本月新增物品数 |
| 即将过期 | 7天内过期物品数 |
| 借出物品 | 当前借出物品数 |

#### 2.7.2 分类统计
- 各分类物品数量分布
- 各分类价值占比
- 分类趋势（新增趋势）

#### 2.7.3 空间统计
- 各空间物品数量分布
- 空间利用率分析
- 空间物品清单

### 2.8 导入导出

#### 2.8.1 数据导出
- 导出格式：JSON / CSV / Excel
- 导出范围：全部/当前筛选结果/指定空间
- 包含物品图片（可选）

#### 2.8.2 数据导入
- 支持JSON/CSV格式导入
- 导入预览和冲突检测
- 批量导入物品

### 2.9 快速添加功能

#### 2.9.1 扫码添加
- 扫描商品条形码自动识别
- 联网获取商品信息（名称/品牌/图片）
- 离线时手动输入

#### 2.9.2 拍照识别
- 拍摄物品照片
- AI识别物品类别（预留）
- 批量添加照片

#### 2.9.3 模板添加
- 常用物品模板库
- 一键快速添加
- 自定义常用模板

## 5. 用户交互流程

### 5.1 添加物品流程

```
1. 用户点击 [+] 按钮
   ↓
2. 进入添加物品页面
   ↓
3. 用户添加物品照片（可选）
   ↓
4. 用户填写物品名称（必填）
   ↓
5. 用户选择所属空间（必填）
   ↓
6. 用户选择分类（必填）
   ↓
7. 用户填写其他信息（可选）
   - 数量、单位
   - 品牌、型号
   - 价格、购买信息
   - 保质期、保修期
   - 标签、备注
   ↓
8. 用户点击"保存物品"
   ↓
9. 验证必填字段
   ↓
10. 保存到 AsyncStorage
    ↓
11. 返回上一页，刷新列表
```

### 5.2 查找物品流程

```
1. 用户进入收纳模块
   ↓
2. 选择查找方式：
   ├─ 方式A：通过空间树导航
   │   ↓
   │   逐级展开空间 → 查看物品
   │
   ├─ 方式B：通过分类快捷入口
   │   ↓
   │   点击分类 → 查看该分类所有物品
   │
   ├─ 方式C：使用搜索
   │   ↓
   │   输入关键词 → 实时显示结果
   │
   └─ 方式D：使用筛选
       ↓
       设置筛选条件 → 应用筛选 → 查看结果
```

### 5.3 借出物品流程

```
1. 用户在物品详情页点击"借出"
   ↓
2. 弹出借出表单
   ↓
3. 用户填写借出信息
   - 借出对象（必填）
   - 借出日期（默认今天）
   - 预计归还日期（可选）
   - 备注（可选）
   ↓
4. 用户确认借出
   ↓
5. 更新物品状态为"借出"
   ↓
6. 记录借还信息
   ↓
7. 返回详情页，状态更新
```

### 5.4 归还物品流程

```
1. 用户在借出物品列表或详情页点击"归还"
   ↓
2. 确认归还
   ↓
3. 更新物品状态为"正常"
   ↓
4. 记录实际归还日期
   ↓
5. 清除借还信息
```
